# 隐私影响评估

## 处理目的

招聘岗位合规审核 Agent 用于辅助平台识别岗位文案中的合规风险，包括就业歧视、入职收费、隐私过度收集、虚假招聘和信息不完整等风险。系统不应采集与岗位审核无关的个人信息。

## 数据类别

| 数据类别 | 示例 | 处理方式 |
| --- | --- | --- |
| 岗位文本 | 标题、描述、职责、要求、福利 | 用于规则扫描、结构化抽取、RAG evidence 匹配 |
| 企业信息 | 公司名称、tenantId、jobPostingId | 用于租户隔离和追踪 |
| 审核结果 | decision、riskLevel、findings、evidenceIds | 用于审核、复核、评估和审计 |
| 操作日志 | actor、operation、resourceId、requestId | 用于敏感操作追踪 |
| 可能出现的个人敏感信息 | 手机号、身份证号、邮箱、银行卡号、微信号、详细地址、验证码 | 保存和导出前默认脱敏 |

## 风险评估

| 风险 | 影响 | 缓解措施 |
| --- | --- | --- |
| 岗位文本夹带个人敏感信息 | 审核日志或模型输入泄露敏感信息 | 统一调用 `redactSensitiveInfo`、`sanitizeAuditLog`、`sanitizeLLMMessages` |
| LLM Provider 外发敏感文本 | 第三方处理范围扩大 | 默认脱敏，真实 Provider 必须经过显式配置和审计 |
| 租户数据越权访问 | 其他 tenant 可见审核记录 | API 强制 `tenantId` 范围校验 |
| 高风险结论不可解释 | 企业难以申诉或修改 | finding 必须绑定 `ruleId` 或 `evidenceId` |
| 数据长期保留 | 超出必要期限 | 通过 `data_retention_jobs` 配置保留期限 |

## 数据主体权利支持

- 删除：`data_deletion_requests` 记录删除请求和执行结果。
- 导出：`privacy_export_requests` 生成脱敏审计记录导出。
- 申诉：人工复核和争议样本流程承接企业或审核员反馈。

## 当前限制

- 数据库适配器的数据删除策略仍需区分硬删除、软删除和审计留痕。
- 真实 LLM 接入前必须完成供应商评估、数据处理协议和密钥管理审查。
- 法规依据库当前为本地维护文本，正式上线前需由法务确认来源和版本。
