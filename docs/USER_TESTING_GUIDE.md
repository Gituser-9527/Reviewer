# User Testing Guide

## 适用对象

- 审核员
- 运营人员
- 合规人员
- 观察员

## 测试步骤

1. 打开 `/beta-launch`，确认自己在使用人员名单中。
2. 查看当前模式：`shadow`、`assist` 或 `limited_enforce`。
3. 使用审核台提交真实脱敏岗位样本。
4. 检查审核结论、风险等级、命中片段、证据和改写建议。
5. 对需要人工判断的样本按 SOP 做复核。
6. 对误杀、漏判、依据错误、体验问题提交反馈。
7. 每日由运营人员生成 Beta 日报。

## 反馈要求

反馈应包含：

- 问题类型
- 严重度
- 相关 auditRunId，若有
- 复现步骤或样本文案摘要
- 期望行为

不要在反馈中粘贴完整手机号、身份证号、银行卡号、验证码或未脱敏候选人信息。
