# 招聘岗位合规审核 Agent 产品需求文档

## 1. 文档信息

- 产品名称：招聘岗位合规审核 Agent
- 阶段：项目初始化 / MVP 规划
- 目标读者：产品、研发、法务/合规、运营、测试与数据评测人员
- 默认市场假设：首期面向中国大陆招聘场景；正式上线前必须由法务确认适用辖区、法规清单和平台规则

## 2. 背景与问题

企业发布岗位信息时，可能出现就业歧视、虚假或误导信息、违规收费、隐私过度收集、薪酬表述不规范、危险或违法工作内容等问题。纯人工审核成本高、口径不一致、难以追溯；纯大模型判断又存在幻觉、结论不稳定和依据不足的问题。

本产品通过“确定性规则优先、模型辅助判断、人工复核兜底”的方式，对岗位信息进行结构化审核，并保留完整审核日志。

## 3. 产品目标

### 3.1 核心目标

对单条岗位信息输出：

1. 审核结论：`PASS`、`BLOCK`、`REVIEW`
2. 风险等级：`NONE`、`LOW`、`MEDIUM`、`HIGH`、`CRITICAL`
3. 命中的违规点及证据片段
4. 法规或平台规则依据
5. 可执行的修改建议
6. 合规改写后的完整岗位文案
7. 可追溯的审核日志

### 3.2 MVP 成功标准

- 高风险规则命中时，系统能够稳定拦截或进入人工复核，不因 LLM 输出而降级为通过
- 每个非通过结论至少关联一个规则 ID 和原文证据片段
- 同一规则版本、模型配置和输入产生可解释、可追踪的结果
- 审核服务异常时默认不放行，返回 `REVIEW` 或明确错误
- 支持人工修改最终结论并记录操作者、原因和时间
- 建立离线评测集，核心高风险类别以召回率为优先指标

### 3.3 非目标

- 不替代律师或法务给出正式法律意见
- MVP 不自动抓取外部法规，不自动更新规则
- MVP 不实现劳动合同、候选人简历或面试过程合规审核
- MVP 不自动发布或下架岗位，只返回审核结果
- MVP 不训练自有基础模型
- MVP 不上线完整 RAG，仅定义接口和数据结构

## 4. 用户与场景

### 4.1 用户角色

| 角色          | 主要诉求                         |
| ------------- | -------------------------------- |
| 企业招聘人员  | 发布前快速发现问题并获得可用改写 |
| 平台审核运营  | 批量审核、查看证据、处理复核队列 |
| 法务/合规人员 | 维护规则依据、审核边界和规则版本 |
| 管理员        | 配置租户、模型、阈值和权限       |
| 审计人员      | 查询完整决策链路和人工操作记录   |

### 4.2 核心用户故事

- 作为招聘人员，我提交岗位标题、职责、要求和薪酬，希望得到明确结论和改写建议。
- 作为审核员，我希望看到命中规则、证据、依据和模型解释，以便快速复核。
- 作为法务人员，我希望规则可版本化、可灰度、可回滚，并能关联法规条款。
- 作为审计人员，我希望确认某次结果使用了哪一版规则、模型和输入。

## 5. 输入范围

### 5.1 必填字段

- 岗位标题 `title`
- 岗位描述原文 `description`

### 5.2 可选结构化字段

- 职责 `responsibilities`
- 任职要求 `requirements`
- 薪酬文本、最低值、最高值、币种和周期
- 工作地点、用工形式、经验、学历、招聘人数
- 企业名称、行业、岗位类别
- 发布渠道、平台规则集、适用地区和语言
- 客户侧业务 ID 与幂等键

缺少关键上下文导致无法判断时，系统应返回 `REVIEW`，并说明缺失项。

## 6. 风险分类

MVP 至少覆盖以下一级类别：

| 类别               | 示例                                                     | 默认处置倾向    |
| ------------------ | -------------------------------------------------------- | --------------- |
| 就业歧视           | 性别、年龄、婚育、民族、地域、户籍、疾病或残障等不当限制 | 高风险拦截/复核 |
| 虚假或误导招聘     | 夸大收入、职责与实际不符、虚构主体                       | 拦截/复核       |
| 违规收费与押金     | 培训费、服装费、押金、保证金                             | 拦截            |
| 隐私与敏感信息     | 发布阶段索取身份证、银行卡、健康等非必要信息             | 拦截/复核       |
| 薪酬与福利         | 范围矛盾、单位缺失、不合理承诺                           | 修改/复核       |
| 非法或危险工作     | 涉诈、色情、赌博、传销、违法兼职等                       | 直接拦截        |
| 未成年人及特殊岗位 | 年龄下限、特殊工种资质或保护要求                         | 拦截/复核       |
| 用工关系表述       | 规避劳动关系、不缴社保、无偿试岗                         | 拦截/复核       |
| 平台内容规范       | 联系方式、外链、导流、格式、敏感词                       | 按平台规则处置  |
| 信息完整性         | 主体、地点、职责或薪酬等关键信息缺失                     | 提示/复核       |

具体规则和依据以已发布规则集为准，分类本身不构成法律结论。

## 7. 审核流程

1. 校验请求、标准化文本并识别适用地区与规则集。
2. 执行确定性 YAML 规则，生成命中项、证据和基础风险分。
3. 对需要语义判断的内容调用 LLM 抽象层，要求输出结构化结果。
4. 聚合规则与模型结果；确定性硬拦截规则拥有最高优先级。
5. 根据风险分、置信度、冲突和信息完整性得到最终结论。
6. 生成修改建议和合规改写；改写不得删除真实岗位核心条件或编造信息。
7. 写入审核结果、执行轨迹和版本快照。
8. `REVIEW` 进入人工复核队列；人工结论覆盖机器结论但不覆盖原始记录。

## 8. 决策规则

### 8.1 结论定义

| 结论     | 定义                                                       |
| -------- | ---------------------------------------------------------- |
| `PASS`   | 未发现需阻止发布的风险，可发布；可附带低风险建议           |
| `BLOCK`  | 命中明确禁止项或风险超过拦截阈值，不应直接发布             |
| `REVIEW` | 证据不足、规则冲突、模型低置信度、关键字段缺失或需专业判断 |

### 8.2 风险等级建议

- `NONE`：无有效命中
- `LOW`：格式、完整性或轻微平台规范问题
- `MEDIUM`：可能造成误导或存在可修正合规风险
- `HIGH`：较明确的歧视、收费、隐私或用工违规风险
- `CRITICAL`：疑似违法业务、人身安全、诈骗等不可发布内容

最终决策不得仅依据模型自报置信度。阈值、优先级和冲突处理见 `RULE_ENGINE.md`。

## 9. 输出要求

每次审核结果必须包含：

- 唯一审核 ID、状态和时间
- 机器结论、当前最终结论、风险等级与数值分
- 命中项列表：规则 ID、类别、严重度、证据、解释、依据
- 建议列表：问题、修改动作、建议文本
- 合规改写文案；无法安全改写时返回 `null` 并说明原因
- 执行元数据：规则集版本、模型提供方/模型、提示词版本、耗时
- 审核日志摘要；敏感调试数据仅限授权用户查看

## 10. 人工复核

- 支持领取、转派、批准、拦截和要求修改
- 必须填写复核原因；结论变更必须记录前后值
- 支持查看原文、机器命中、依据和历史处理
- 法务可将误报/漏报标记为评测样本候选
- 人工处理 SLA、队列优先级和通知方式在 MVP 可先配置为运营流程

## 11. 非功能需求

- 安全：租户隔离、最小权限、日志脱敏、传输与静态加密
- 可用性：LLM 不可用时确定性规则仍可执行，并默认进入复核而非放行
- 可观测性：请求、规则、模型、数据库各阶段有指标和关联 ID
- 可审计性：关键记录追加写，规则和模型配置可追溯
- 性能：MVP 单条同步审核 P95 目标不高于 5 秒；不含人工复核
- 扩展性：支持新增司法辖区、平台规则集、模型提供方和异步批量审核
- 隐私：明确数据保留期；禁止将未授权岗位数据用于模型训练

## 12. 验收标准

- 可通过 REST API 创建审核并查询结果
- 至少实现一套经法务确认的 MVP 规则集及版本管理流程
- 硬拦截规则无法被模型结论覆盖
- 所有 `BLOCK`/`REVIEW` 结果包含可定位到原文的证据
- 改写结果通过二次规则检查后才可返回为“建议合规文案”
- 可完成人工复核并查询完整历史
- 评测报告能按风险类别展示召回率、精确率和决策混淆矩阵

## 13. 待决策事项

- 首发司法辖区和具体平台规则来源
- 是否允许客户自定义规则覆盖平台规则，以及优先级边界
- 岗位原文、模型提示和响应的保留周期
- MVP 是否开放批量导入与 Webhook
- 人工复核团队、SLA 与升级路径
- LLM 供应商的数据驻留、零保留和合规要求
