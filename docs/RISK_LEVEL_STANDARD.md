# 风险等级标准

本文档定义招聘岗位合规审核中的风险等级口径，供 Agent 输出、人工复核和评估标注共同使用。

## 1. 风险等级

| 等级 | 含义 | 建议动作 |
| ---- | ---- | -------- |
| `NONE` | 未发现当前规则可识别风险 | 可通过 |
| `LOW` | 轻微不规范或提示性风险 | 可通过并提示 |
| `MEDIUM` | 存在可修改风险 | 允许发布需警示或要求修改 |
| `HIGH` | 存在较高合规风险或证据不足但影响较大 | 人工复核 |
| `CRITICAL` | 存在明确高风险命中，如收费、押金、显性歧视 | 建议拦截 |

## 2. 常见等级示例

### DISCRIMINATION

- `CRITICAL`：明确限制性别、婚育、民族等与岗位无关条件。
- `HIGH`：年龄、学历等限制疑似不合理，但存在岗位例外可能。
- `MEDIUM`：偏好性表达不清，需要修改为能力要求。

### FEE_DEPOSIT

- `CRITICAL`：入职收费、押金、服装费、培训贷、担保金等。
- `HIGH`：费用表达不清，疑似向劳动者收费。

### PRIVACY

- `HIGH`：要求身份证、银行卡、家庭详细信息等超出招聘必要范围。
- `MEDIUM`：要求联系方式但目的和保护措施不清。

### FALSE_OR_MISLEADING

- `HIGH`：明显夸大高薪、承诺不实收益或职位内容与实际不符。
- `MEDIUM`：薪资、福利、晋升表述模糊但可修改。

### INCOMPLETE_INFORMATION

- `MEDIUM`：缺少岗位职责、地点、薪资或用工类型等关键信息。
- `LOW`：个别非关键字段缺失。

## 3. 决策映射建议

| 风险等级 | Agent 决策 |
| -------- | ---------- |
| `NONE` | `PASS` |
| `LOW` | `PASS` 或 `ALLOW_WITH_WARNING` |
| `MEDIUM` | `ALLOW_WITH_WARNING` |
| `HIGH` | `MANUAL_REVIEW` |
| `CRITICAL` | `REJECT` |

## 4. 标注注意事项

- 风险等级应基于原文和依据，不基于企业历史印象。
- evidence 不足时，不应直接给出确定性高风险结论。
- 多风险混合时，整体风险等级取最高风险。
- 人工裁决可以调整 Agent 风险等级，但必须写明理由。
