# Data Leak Response

## 触发条件

- 日志、反馈、导出报告中出现完整手机号、身份证号、银行卡号、验证码或 API Key。
- LLM 输入包含未脱敏敏感信息。
- 错误响应暴露数据库连接串、内部 Prompt 或供应商原始响应。

## 处置

1. 立即开启 `force_manual_review`，必要时暂停相关入口。
2. 创建 P0/P1 事故记录。
3. 标记和隔离相关日志、反馈或报告。
4. 执行数据删除或脱敏修复。
5. 审查调用链是否绕过 `packages/core/src/security/`。
6. 生成复盘报告，并增加脱敏测试。

## 对外沟通

只使用已确认事实；不要猜测影响范围。涉及客户数据时由合规负责人统一沟通。
