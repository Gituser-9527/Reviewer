# Reviewer Training Manual

## 审核员职责

审核员负责判断 Agent 输出是否适合用于平台审核流程，并补充统一标签。审核员不是法律裁判，应使用审慎表述。

## 基本流程

1. 阅读岗位原文、Agent 结论、风险等级和 findings。
2. 检查命中片段是否真实出现在原文中。
3. 检查依据是否与风险类别相关。
4. 判断人工最终结论：`APPROVE`、`REJECT` 或 `REQUEST_REVISION`。
5. 选择反馈类型，写明理由。
6. 只有标签结构明确、争议已处理的样本才进入评估集。

## 反馈原则

- `VALID_RESULT`：Agent 结果有效。
- `FALSE_POSITIVE`：Agent 误杀，人工认为不应判为风险。
- `FALSE_NEGATIVE`：Agent 漏判，人工发现风险。
- `WRONG_EVIDENCE`：依据不相关、过期或无法支撑解释。
- `BAD_REWRITE`：改写文案仍包含风险表达。

## 禁止事项

- 不要在反馈里粘贴完整手机号、身份证号、银行卡号、验证码。
- 不要输出“企业已经违法”等绝对法律结论。
- 不要把个人主观偏好标成规则问题。
- 不要把证据不足的样本直接标成确定违法。
