# RAG 设计预案

## 1. 阶段定位

MVP 不依赖 RAG 做最终决策，但预留稳定接口、引用对象和 pgvector 数据模型。RAG 的目标是为语义判断提供经审核的法规、平台规则、解释材料和历史案例依据，而不是让模型自由搜索并生成法律结论。

## 2. 使用场景

- 根据司法辖区、平台和审核时间检索有效法规/规则条款
- 为命中项补充可核验的依据摘要和来源
- 辅助判断规则例外、特殊岗位或新型表达
- 为人工审核员提供相似已裁决案例
- 支持法规变化影响分析和规则维护

MVP 禁止将未经审核的互联网内容作为自动拦截依据。

## 3. 组件接口

```ts
interface ComplianceKnowledgePort {
  retrieve(query: KnowledgeQuery): Promise<KnowledgeResult[]>;
  getAuthoritySnapshot(id: string, asOf: string): Promise<AuthoritySnapshot | null>;
}
```

`KnowledgeQuery` 应包含：文本、司法辖区、平台、语言、审核时间、风险类别、topK 和权限范围。

`KnowledgeResult` 应包含：文档/分块 ID、authority ID、标题、条款路径、摘要、来源 URL、版本、生效期、检索分数和内容哈希。

业务层只依赖接口；MVP 可提供返回空结果的实现。

## 4. 知识来源治理

来源优先级：

1. 官方法律法规和政府部门公开文件
2. 已批准的平台规则
3. 内部法务解释和操作手册
4. 已完成人工裁决且允许复用的匿名案例

每个文档必须记录来源、抓取/上传人、审核人、版本、生效日期、失效日期、司法辖区、内容哈希和授权范围。来源不明确、已失效或待审核的文档不得进入生产检索集合。

## 5. 摄取与索引流程

```mermaid
flowchart LR
  Source["受信来源"] --> Parse["解析与结构保留"]
  Parse --> Validate["完整性/版本校验"]
  Validate --> Approve["法务审批"]
  Approve --> Chunk["按条款分块"]
  Chunk --> Embed["生成 Embedding"]
  Embed --> Index["PostgreSQL + pgvector"]
  Index --> Eval["检索评测"]
  Eval --> Publish["发布索引版本"]
```

- 法规按章、节、条分块，避免固定字符切分破坏条款边界
- 每个 chunk 保留父级路径与相邻块关系
- 表格、附件和修订说明需结构化处理
- 同一来源新版本上线后，旧版本保留但按审核时间过滤

## 6. 检索策略

建议采用混合检索：

- 元数据硬过滤：租户权限、司法辖区、平台、生效时间、状态
- PostgreSQL 全文/BM25 类关键词检索
- pgvector 语义检索
- 可选 reranker 对候选条款重排

最终分数由关键词、向量、来源权威度、时效性和类别匹配组合。不得仅按向量相似度选择法规依据。

## 7. 引用与生成约束

- 模型只能引用检索结果中的 authority ID 和条款
- 输出必须区分原文引述、系统摘要和模型解释
- 引用必须带版本和审核时间；过期依据标记为不可用于当前结论
- 检索无结果或结果冲突时返回“依据不足”并进入 `REVIEW`
- 不允许模型伪造 URL、条款号或法规名称

## 8. 安全与多租户

- 公共法规与租户私有政策分 collection/metadata 权限隔离
- 检索查询和结果受租户过滤，数据库层再做防护
- 案例在索引前去标识化，禁止嵌入候选人个人信息
- Embedding Provider 也必须满足数据处理与保留要求
- 对提示注入文本按不可信数据处理，不执行文档内指令

## 9. 版本与缓存

- 记录 `knowledge_index_version`、embedding 模型和维度
- 任何审核保存所使用的检索结果快照或稳定引用
- Embedding 模型变化需要重建独立索引，不原地混用向量
- 缓存键包含查询哈希、权限范围、索引版本和审核时间

## 10. 评测指标

- `Recall@K`：正确依据是否进入候选
- `MRR/nDCG`：正确依据排序质量
- Citation precision：最终引用是否真实支持结论
- 时效正确率：是否过滤未生效/已失效版本
- 无依据拒答率：找不到依据时是否避免编造
- 检索延迟和每次查询成本

## 11. 分阶段实施

### Phase 0：MVP 预留

- 定义 Port、引用 DTO、数据库预留和空实现
- 审核结果可关联 authority 快照

### Phase 1：受控法规库

- 人工上传与审批
- 混合检索、时间过滤和引用展示
- 建立检索黄金集

### Phase 2：自动更新辅助

- 官方来源变更检测、差异报告和人工审批
- 规则影响分析，不自动发布规则

### Phase 3：案例与高级推理

- 匿名裁决案例检索
- reranker、多跳检索和冲突依据提示
