# 上线合规评审

## 评审目标

在进入真实岗位数据封闭试运行或生产流量前，确认招聘岗位合规审核 Agent 具备安全、隐私、可解释、可审计和可回滚能力。

## 评审结论分级

- `ready`：关键安全与合规门禁通过，可以进入受控试运行。
- `needs_attention`：存在需要修复或人工确认的问题，不建议扩大流量。
- `blocked`：存在阻断项，不建议上线。

## 自动检查

运行：

```bash
npm run build
npm test
npm run eval
```

启动 API 后调用：

```http
GET /api/security/launch-check/report
```

报告必须覆盖：

- 是否保存个人敏感信息明文
- LLM 输入是否默认脱敏
- 审核日志是否配置保留期限
- 是否支持数据删除
- 是否支持导出审计记录
- 是否记录模型、规则、知识库版本
- 是否可以解释每个拦截原因
- 是否有人工申诉和复核入口
- 是否有安全事件处理流程
- 是否有权限隔离

## 人工确认

上线前必须由产品、法务、算法/规则、后端和安全负责人确认：

- 当前规则库与平台政策一致。
- 本地知识库 evidence 来源、版本和引用方式可接受。
- 数据保留期限满足业务、法规和平台要求。
- 真实 LLM Provider 的数据处理边界已审批。
- 灰度、回滚、人工复核和申诉路径可执行。

## 安全事件处理流程

1. 发现疑似敏感信息泄露、越权访问、错误拦截大幅升高或模型异常。
2. 立即暂停相关 rollout 或切回 stableVersion。
3. 导出相关审计日志和 security check result。
4. 通知安全、法务、平台运营和技术负责人。
5. 修复后补充回归测试和评估样本。
6. 复盘并更新本评审文档或 ADR。

## 当前 MVP 限制

- 数据删除在内存存储中已闭环，PostgreSQL 生产仓储仍需补充可审计删除实现。
- LLM Provider 默认不调用真实外部模型，真实接入前需完成独立安全评审。
- 法规和平台规则 evidence 当前为人工维护文本，正式生产前需要法务确认。
