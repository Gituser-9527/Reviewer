# 人工审核 SOP

本文档规定人工审核员如何复核招聘岗位合规审核 Agent 的输出，确保审核结论、反馈类型和后续评估样本口径一致。

## 1. 审核目标

人工审核的目标不是替代法律裁判，而是在平台发布流程中确认岗位文案是否存在需要拦截、修改或继续复核的合规风险。

审核员应判断：

- Agent 是否识别了关键风险
- Agent 的风险类别是否正确
- Agent 的风险等级是否合理
- Agent 引用的 evidence 是否支撑 finding
- 修改建议是否可执行
- 是否需要进入争议样本池或规则改进流程

## 2. 审核顺序

1. 阅读岗位标题、公司名称、岗位描述和结构化字段。
2. 查看 Agent 的最终结论、风险等级和摘要。
3. 逐条检查 findings：
   - 命中文本是否出现在原文
   - 风险类别是否匹配
   - 严重程度是否合理
   - ruleId / evidenceId 是否存在
4. 检查 evidence：
   - 是否与 finding category 相关
   - 是否为人工维护或可信来源
   - 是否存在引用错误或过度解释
5. 给出人工标注：
   - 最终结论
   - 风险类别
   - 风险等级
   - 反馈类型
   - 简短理由
6. 如与其他审核员不一致，样本进入争议池。
7. 高级审核员对争议样本作最终裁决。

## 3. 人工最终结论

| 结论 | 使用场景 |
| ---- | -------- |
| `APPROVE` | 岗位可发布，未发现需要阻断的明显风险 |
| `REJECT` | 存在明确高风险内容，建议拦截 |
| `REQUEST_REVISION` | 存在可修改风险，建议企业修改后重新提交 |

## 4. 多人标注规则

- 同一条样本至少支持多名审核员独立标注。
- 审核员不得先查看其他审核员的标签再标注。
- 多人标注使用统一字段：结论、风险类别、风险等级、反馈类型、理由。
- 当最终结论、风险类别或风险等级不一致时，系统应标记为争议样本。
- 争议样本不得直接进入冻结评估集，必须先由高级审核员裁决。

## 5. 争议处理

争议样本进入 `disputed_cases` 后，由高级审核员处理：

1. 查看所有审核员标签。
2. 检查 Agent finding、evidence 和原文。
3. 确定统一最终标签。
4. 写明裁决理由。
5. 决定是否进入评估集、规则改进或知识库修订流程。

## 6. 输出要求

人工反馈应避免绝对法律结论。推荐表达：

- “存在合规风险”
- “建议拦截”
- “建议修改后重新提交”
- “当前 evidence 不足，建议复核”

禁止表达：

- “企业已经违法”
- “构成犯罪”
- “必然承担法律责任”

## 7. 质量要求

- 高风险标签必须能对应原文片段、ruleId 或 evidenceId。
- 反馈理由应简洁、可复查，不写无关主观评价。
- 发现误杀/漏判必须选择对应反馈类型。
- 发现 evidence 引用错误必须选择 `WRONG_EVIDENCE`。
